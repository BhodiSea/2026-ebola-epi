# Drizzle cheat-sheet (query layer only)

Drizzle is the **typed query layer** in this repo. The schema source of
truth is raw SQL under `supabase/migrations/`. Drizzle schema files in
`packages/db/src/schema/` are aligned to the SQL — generated by
`supabase gen types typescript` (for row types) + handwritten Drizzle table
defs (for query DSL).

`drizzle-kit push` and `drizzle-kit generate` are **banned** in this repo.

## drizzle.config.ts (target)

```ts
import { defineConfig } from 'drizzle-kit';

export default defineConfig({
  schema: './src/schema',
  dialect: 'postgresql',
  dbCredentials: { url: process.env.DATABASE_URL! },
  extensionsFilters: ['postgis'], // surface postgis types in introspection
  schemaFilter: ['public', 'private'],
});
```

`extensionsFilters: ['postgis']` is required for PostGIS introspection. We
introspect to verify the Drizzle schema matches the SQL, NOT to generate
migrations.

## Branded IDs

```ts
import { z } from 'zod';

export const SitrepId = z.string().uuid().brand<'SitrepId'>();
export type SitrepId = z.infer<typeof SitrepId>;

export const SourceQuoteId = z.string().uuid().brand<'SourceQuoteId'>();
export type SourceQuoteId = z.infer<typeof SourceQuoteId>;
```

In Drizzle table defs cast at the boundary:

```ts
import { pgTable, uuid, timestamp, text } from 'drizzle-orm/pg-core';

export const sourceQuotes = pgTable('source_quotes', {
  id: uuid('id').primaryKey().defaultRandom().$type<SourceQuoteId>(),
  sitrepId: uuid('sitrep_id').notNull().$type<SitrepId>(),
  text: text('text').notNull(),
  sha256: text('sha256').notNull(),
  charStart: integer('char_start').notNull(),
  charEnd: integer('char_end').notNull(),
  createdAt: timestamp('created_at', { withTimezone: true, mode: 'date' }).defaultNow().notNull(),
});
```

Use `mode: 'date'` on timestamps — picks Date in TS without needing
ad-hoc `new Date(s)` callsites.

## PostGIS

```ts
import { pgTable, uuid, geometry } from 'drizzle-orm/pg-core';

export const healthZones = pgTable('health_zones', {
  id: uuid('id').primaryKey().defaultRandom().$type<ZoneId>(),
  centroid: geometry('centroid', { type: 'point', srid: 4326 }).notNull(),
  // boundaries stored separately; this is a centroid for quick distance queries
});
```

- **SRID 4326 always.** Use `geometry`, not `geography`, unless distances
  matter on a global scale.
- Boundary polygons live in a materialised view simplified by zoom level
  (`zone_geom_z6`, `zone_geom_z10`). Refreshed nightly.

## pgvector

```ts
import { pgTable, uuid, text, vector, index } from 'drizzle-orm/pg-core';
import { cosineDistance } from 'drizzle-orm';

export const sourceQuoteEmbeddings = pgTable(
  'source_quote_embeddings',
  {
    id: uuid('id').primaryKey().defaultRandom(),
    quoteId: uuid('quote_id').notNull().$type<SourceQuoteId>(),
    embedding: vector('embedding', { dimensions: 1536 }).notNull(),
  },
  (t) => ({
    cosineIdx: index('quote_embeddings_cosine_idx')
      .using('hnsw', t.embedding.op('vector_cosine_ops')),
  }),
);

// query
import { sql } from 'drizzle-orm';

const similar = await db
  .select({
    id: sourceQuoteEmbeddings.id,
    distance: cosineDistance(sourceQuoteEmbeddings.embedding, query),
  })
  .from(sourceQuoteEmbeddings)
  .orderBy(cosineDistance(sourceQuoteEmbeddings.embedding, query))
  .limit(10);
```

- HNSW max 2000 dimensions. `text-embedding-3-small` (1536) fits.
- Cosine for normalised OpenAI / Voyage embeddings. L2 for some others.
- HNSW expects the graph in RAM; size your Supabase tier accordingly.

## Forbidden

- `drizzle-kit push` / `drizzle-kit generate` — migrations are raw SQL.
- `db.execute(sql\`…\`)` for migrations — that's a query escape hatch only.
- Importing Drizzle schema in `packages/shared/` — it's not runtime-agnostic
  (uses Node-only `pg-core`). Branded IDs only in `shared/`.
- Querying without `enable row level security` confirmation in the matching
  migration; Drizzle queries are not exempt from RLS.

## After every migration

```bash
pnpm db:types        # supabase gen types typescript → packages/db/src/types.gen.ts
```

Commit `types.gen.ts`. Reviewers should see schema changes in the diff.
